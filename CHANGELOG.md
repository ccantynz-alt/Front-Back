# Crontech Changelog

This file is generated by `scripts/generate-changelog.ts`. It collects
commits since the previous boundary marker, hands them to Claude Haiku,
and writes three sections per release:

- **User notes** — polite release notes for end users.
- **Dev notes** — technical changes, breaking changes, and migration steps.
- **Social snippet** — a single tweet-length announcement.

Each release is bracketed by a boundary marker that the next run keys
off when picking up where the last one left off:

```
<!-- changelog-boundary:YYYY-MM-DD -->
```

Do not edit boundary markers by hand. The generator manages them.
